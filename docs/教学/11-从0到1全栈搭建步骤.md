## 阶段 2：数据库 schema + 实体 + Mapper

### 2.1 设计表

业务："员工 + 部门 + 考勤"。

`schema.sql`：

```sql
DROP TABLE IF EXISTS attendance_record;
DROP TABLE IF EXISTS employee;
DROP TABLE IF EXISTS department;

CREATE TABLE department (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    code VARCHAR(32) NOT NULL UNIQUE,
    name VARCHAR(64) NOT NULL,
    status INT NOT NULL DEFAULT 1,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE employee (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(40) NOT NULL,
    phone VARCHAR(20) NOT NULL UNIQUE,
    email VARCHAR(120),
    department_id BIGINT NOT NULL,
    position VARCHAR(60) NOT NULL,
    hire_date DATE NOT NULL,
    salary DECIMAL(12, 2) NOT NULL,
    status INT NOT NULL DEFAULT 1,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE attendance_record (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    employee_id BIGINT NOT NULL,
    work_date DATE NOT NULL,
    check_in_time TIMESTAMP,
    check_out_time TIMESTAMP,
    status INT NOT NULL DEFAULT 1,
    note VARCHAR(255),
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT uk_attendance_employee_date UNIQUE (employee_id, work_date)
);
```

`data.sql` 写几条初始化数据，启动就能用。

### 2.2 写 Entity

每张表一个 Java 类：`Department`、`Employee`、`AttendanceRecord`。字段类型按 [04-MyBatis](./04-MyBatis%20与数据库操作.md) 的对照表选。

### 2.3 写 Mapper + XML

接口里声明方法，XML 写 SQL。先实现 CRUD 的基本款（findById、insert、update、disable），再补查询专用（search、count、findByPhone）。

完成此阶段，能在 H2 控制台（<http://localhost:8080/h2-console>，JDBC URL `jdbc:h2:mem:company_demo`）里看到 3 张表 + 初始数据。

## 阶段 3：Service + DTO + 校验

每个聚合根（员工、部门、考勤）一个 Service。代码写法见 [03-Spring Boot](./03-Spring%20Boot%20框架手把手.md) 与 [05-后端分层](./05-后端分层架构与统一响应.md) 章节。

### 关键设计点

- 创建员工时校验部门存在
- 修改员工 / 离职员工时校验 ID 存在
- 上班 / 下班打卡用 "INSERT ... ON CONFLICT (employee_id, work_date) UPDATE ..." 思路（H2 / MySQL 都支持类似 upsert）

## 阶段 4：Controller + 统一响应 + 异常

写 `EmployeeController`、`DepartmentController`、`AttendanceController`。

`common/` 下面准备：

- `ApiResponse<T>`：统一响应包装
- `PageResult<T>`：分页结构
- `BusinessException`：业务异常

## 阶段 6：给前端开门 —— CORS

新建 `common/CorsConfig.java`：

```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowedOriginPatterns("http://localhost:*", "http://127.0.0.1:*")
                .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
                .allowedHeaders("*")
                .allowCredentials(true)
                .maxAge(3600);
    }
}
```

## 阶段 8：接入 Element Plus + Router + Pinia

`main.ts` 把三件套挂上去：

```ts
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import ElementPlus from 'element-plus'
import zhCn from 'element-plus/dist/locale/zh-cn.mjs'
import 'element-plus/dist/index.css'
import App from './App.vue'
import router from './router'
import './assets/main.css'

const app = createApp(App)
app.use(createPinia())
app.use(router)
app.use(ElementPlus, { locale: zhCn })
app.mount('#app')
```

写一个最简单的 `App.vue`：

```vue
<template>
  <RouterView />
</template>
```

加 `router/index.ts`、`stores/department.ts`、`utils/request.ts`、`api/*.ts`、`types/index.ts`，结构见仓库。

## 阶段 9：三个页面

实现：

1. `MainLayout.vue`：左侧菜单 + 顶部 + 主内容
2. `EmployeeListView.vue`：分页 + 查询 + 新增 / 编辑 / 离职
3. `DepartmentListView.vue`：列表 + 新增
4. `AttendanceView.vue`：选员工 → 上下班打卡 → 查询历史

每个页面都用：

- Element Plus 组件
- Pinia store（部门数据共享）
- axios 封装的 `api/xxx.ts`
- TS 类型定义在 `types/index.ts`
