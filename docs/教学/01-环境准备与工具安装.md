# 01 环境准备与工具安装

在写代码之前，先把"工具箱"搭好。本章覆盖：

1. JDK（Java 开发工具包）
2. Maven（Java 的依赖管理 / 构建工具）
3. Node.js（前端运行环境）
4. pnpm（前端包管理器，比 npm 快）
5. MySQL（数据库，可选）
6. Git（版本控制工具）
7. IDE：IntelliJ IDEA + VS Code

> 阅读这章不需要任何前置知识。只要你能在 Windows / macOS 里安装软件就行。

## 1. JDK：Java 开发工具包

Java 程序运行需要 JDK（Java Development Kit）。本项目要求 **JDK 11 或更高**。

### Windows 安装

推荐用 Adoptium / Temurin（社区版 OpenJDK，永久免费）：

1. 打开 <https://adoptium.net/temurin/releases/>
2. Operating System 选 Windows，Architecture 选 x64，Package Type 选 JDK，Version 选 17 LTS（11 也可以）
3. 下载 `.msi` 安装包，双击安装，下一步到底
4. 安装时勾选 "Set JAVA_HOME variable" 和 "Add to PATH"，安装好就不用手动配环境变量了

验证：

```powershell
# Win + R 输入 cmd 打开命令提示符
java -version
javac -version
```

看到类似这样就成功：

```text
openjdk version "17.0.10" 2024-01-16
OpenJDK Runtime Environment Temurin-17.0.10+7
OpenJDK 64-Bit Server VM Temurin-17.0.10+7
```

### macOS 安装

推荐用 Homebrew：

```bash
brew install --cask temurin@17
```

### 常见坑

- **报 `'java' 不是内部或外部命令`**：说明 PATH 没配。打开"系统属性 → 环境变量"，在 `Path` 里加入 `JAVA_HOME\bin`。
- **公司里有 JDK 8 项目，怎么共存？** 在系统装多个版本，用 `JAVA_HOME` 切换；或者用 [SDKMAN!](https://sdkman.io/)（Linux/macOS）。

## 2. Maven：项目构建与依赖管理

Java 项目里几乎所有"第三方包"都靠 Maven 拉取。

### Windows 安装

1. 下载：<https://maven.apache.org/download.cgi>，选 `apache-maven-3.9.x-bin.zip`
2. 解压到 `D:\dev\apache-maven-3.9.x`
3. 配环境变量：
   - 新增 `MAVEN_HOME=D:\dev\apache-maven-3.9.x`
   - 在 `Path` 里追加 `%MAVEN_HOME%\bin`
4. 验证：

```powershell
mvn -v
```

### 配置国内镜像（强烈推荐）

否则下载依赖会非常慢。打开 `MAVEN_HOME\conf\settings.xml`，找到 `<mirrors>` 节点，加：

```xml
<mirror>
  <id>aliyun-maven</id>
  <mirrorOf>*</mirrorOf>
  <name>阿里云公共仓库</name>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

> 如果用 IDEA 自带 Maven，记得在 `设置 → Build, Execution, Deployment → Build Tools → Maven` 里把 User settings file 指到你修改后的 `settings.xml`。

## 3. Node.js：前端运行环境

本项目前端用 Vue3 + Vite，要求 **Node.js 18+**（推荐 20 LTS）。

### Windows / macOS 安装

直接到 <https://nodejs.org/zh-cn/> 下载 LTS 版本，一路下一步。

验证：

```bash
node -v
npm -v
```

### 切换源（强烈推荐）

```bash
npm config set registry https://registry.npmmirror.com
```

### 多版本共存：nvm

公司里不同项目可能要不同 Node 版本。建议用 [nvm-windows](https://github.com/coreybutler/nvm-windows) 或 [nvm](https://github.com/nvm-sh/nvm)（macOS/Linux）。

```bash
nvm install 20
nvm use 20
```

## 4. pnpm：更快的包管理器

pnpm 是 npm 的替代品，安装快、磁盘占用小，本项目用它。

```bash
npm install -g pnpm
pnpm -v
```

## 5. MySQL：可选

新手可以**先跳过**，项目默认用 H2 内嵌数据库，启动就能用，不需要装 MySQL。等你想真实把数据持久化、用 Navicat 看表，再装 MySQL 也来得及。

### Windows 装 MySQL

1. 下载 MySQL Installer：<https://dev.mysql.com/downloads/installer/>
2. 选 "Custom"，只装 "MySQL Server" + "MySQL Workbench"
3. 安装时设 `root` 密码，记牢

### 用 Docker 一行起 MySQL（推荐）

如果电脑装了 Docker Desktop：

```bash
docker run -d --name mysql8 \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=root123 \
  -e MYSQL_DATABASE=company_demo \
  mysql:8.0
```

启动后用任何客户端（Navicat / DataGrip / DBeaver）连：

- Host: `127.0.0.1`
- Port: `3306`
- User: `root`
- Password: `root123`
- Database: `company_demo`

## 6. Git：版本控制工具

Git 是协作开发的"必备技能"，详细用法见 [10-Git 与 GitHub 小白教程](./10-Git%20与%20GitHub%20小白教程.md)。

### Windows 安装

下载 <https://git-scm.com/download/win>，一路下一步即可（默认设置就行）。

验证：

```powershell
git --version
```

第一次用要设置一下身份（这个会写到每次提交里）：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

## 7. IDE：写代码用什么

### IntelliJ IDEA（写 Java 后端）

- 社区版（Community）免费够用：<https://www.jetbrains.com/idea/download/>
- 学生 / 教师可以免费申请专业版（Ultimate）

第一次打开项目：

1. `File → Open` 选项目根目录（含 `pom.xml`）
2. 等右下角"Indexing"+"Maven importing"跑完
3. 在 `CompanyDemoApplication.java` 右键 → Run

常用快捷键（默认 Win/Linux）：

| 快捷键 | 作用 |
| --- | --- |
| `Alt+Insert` | 自动生成 Getter/Setter/构造方法 |
| `Ctrl+Alt+L` | 格式化代码 |
| `Ctrl+B` | 跳到定义 |
| `Ctrl+Alt+B` | 跳到实现 |
| `Shift+Shift` | 全局搜索（Search Everywhere） |
| `Ctrl+R` | 替换 |
| `Ctrl+E` | 最近打开的文件 |
| `Shift+F10` | 运行 |
| `Shift+F9` | 调试 |

### VS Code（写前端 Vue + TS）

下载：<https://code.visualstudio.com/>。装这些插件：

- **Vue - Official**（旧名 Volar）：Vue3 + TS 必装
- **ESLint**：代码风格检查
- **Prettier**：格式化
- **Git Graph**：图形化看 Git 历史
- **GitLens**：每行代码的提交记录

> 注意：装了 **Vue - Official** 之后，要在设置里把 "Vue: Server: Hybrid Mode" 关掉，或者把 TypeScript 内置插件 Volar 的 takeover 模式打开（看插件说明）。

## 8. 一键自检脚本

跑一遍这些命令，全部输出版本就说明环境 OK：

```bash
java -version
javac -version
mvn -v
node -v
npm -v
pnpm -v
git --version
```

## 小结

环境准备本身没有"知识点"，但是 80% 的初学者卡在这里就放弃了。请一定要：

- **耐心**：每个工具都要装，少一个都跑不起来
- **记路径**：JDK / Maven / Node 装在哪里，自己写在便签上
- **改镜像源**：Maven、npm 都切到国内源
- **不要怕版本不一样**：JDK 11 / 17，Node 18 / 20，差不多就行
